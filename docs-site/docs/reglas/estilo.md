# Estilo de escritura

## Reglas drásticas (ERROR)

### Doble espacio

!!! danger "ERROR"
    El sistema detecta `  ` (dos espacios consecutivos) como error. Es un problema tipográfico que rompe la alineación.

    Solución: Ctrl+H → reemplazar `  ` por ` ` en todo el documento.

### Enters múltiples para simular Salto de Página

!!! danger "ERROR"
    Más de 3 Enters consecutivos para empujar contenido a la siguiente página → ERROR.

    Solución: usar **Ctrl + Enter** (Insertar → Salto de página).

## Reglas de redacción (sugerencias)

### Oraciones muy largas

Oraciones de **más de 50 palabras** son difíciles de leer. El sistema las marca como advertencia y sugiere dividirlas.

### Párrafos sin verbos

Párrafos largos sin verbos suelen indicar errores de pegado o fragmentos incompletos.

### Inicio con minúscula

Cada párrafo debe comenzar con mayúscula.

### Puntuación

- No espacios antes de signos (espacio antes de punto o coma → error)
- Espacio obligatorio después de signo (`hola,como estás` → falta espacio)
- Sin punto final en párrafos largos → advertencia

## Reglas anti-evasión

### Texto en cuadros de texto

!!! warning "Sugerencia firme"
    Poner texto dentro de cuadros de texto (text boxes) para que Turnitin no lo procese NO ES ACEPTABLE.

    Solución: transcribir el contenido a párrafos normales.

### Capturas de pantalla con texto

!!! warning "Sugerencia firme"
    Las imágenes grandes con texto (capturas de pantalla) sin etiqueta de "Figura N" se sugieren transcribir.

    Si la imagen es una figura legítima (gráfico, fotografía, esquema), debe tener:
    
    ```
    Figura 5
    Título descriptivo en cursiva
    [ IMAGEN ]
    Nota: descripción adicional.
    ```
